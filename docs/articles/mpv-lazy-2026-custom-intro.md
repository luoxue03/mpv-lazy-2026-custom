基于 `hooke007` 大佬的 `mpv-lazy / mpv_PlayKit` 项目修改。

主要目标是：

> 尽量开箱即用，让不会自己写配置的朋友，也可以比较方便地使用 MPV 看番、补帧、超分和调整画面效果。

## 1、这是什么？

先简单说一下。

`MPV` 是一个非常强大的播放器，但原版界面和配置都比较“硬核”。

`mpv-lazy` 则是基于 MPV 做好的 Windows 懒人包，里面已经整合了大量配置、脚本、着色器、滤镜和中文菜单。

而我这个版本，是在 `mpv-lazy 2026` 的基础上，又继续做了一些自定义整合。

主要增加和整理了这些内容：

- RIFE 补帧模型
- AnimeJaNai / UAI 等超分模型
- 补帧 + 超分组合
- 动画常用着色器组合
- RIFE 自定义参数菜单
- RIFE 性能测试报告
- 更适合自己使用习惯的菜单分类

如果你之前用过我旧版的 2025 包，可以把这次理解成：

> 2026 新版底座 + 旧版自定义功能整理迁移 + 一些新功能补充。

插件相关内容，比如在线视频、弹幕、AI 字幕、最近播放、清晰度选择等，准备放到下一篇单独讲。这篇先讲最基础的播放、补帧、超分和着色器。

## 2、适合什么人？

这个整合包比较适合：

- 平时主要看动画
- 想试试补帧和超分，但不想从零写配置
- 有 N 卡，尤其是 RTX 系列显卡
- 能接受偶尔看一下说明、自己切换模型
- 想要比普通播放器更高一点的可玩性

不太适合：

- 只想双击播放，完全不想折腾
- 显卡性能较弱
- 不想看任何说明文档
- 主要看真人电影，并且片源本身已经很清晰
- 对稳定性要求极高，不接受任何脚本或滤镜报错

MPV 很强，但它不是那种“所有东西都自动帮你选好”的播放器。

如果你愿意稍微花点时间理解菜单，它会很好用。

## 3、安装方式

下载完成后，将压缩包解压到一个固定目录。

建议不要放在中文路径，也不要放在权限很严格的目录，比如系统盘某些受保护文件夹。

解压后，大致会看到这些内容：

```text
mpv-lazy
├── installer
├── portable_config
├── mpv.exe
├── mpv.com
├── yt-dlp.exe
└── ...
```

其中比较重要的是：

```text
installer/mpv-install.bat
installer/mpv-uninstall.bat
```

`mpv-install.bat` 是注册播放器用的。

用管理员身份运行后，可以把 MPV 注册到系统里，这样你就可以在 Windows 里把视频默认打开方式设置为 MPV。

`mpv-uninstall.bat` 则是反注册。

注意：

> 注册完成后，不建议随便移动整个 MPV 文件夹。  
> 如果你移动了目录，最好重新运行一次注册脚本。

因为系统文件关联记录的是播放器路径，路径变了就可能打不开。

## 4、基础使用

安装完成后，可以直接双击 `mpv.exe` 打开播放器，也可以双击视频文件播放。

常用操作大概是：

- 空格：暂停 / 播放
- 鼠标右键：打开菜单
- `f`：全屏
- `Shift+i`：显示详细播放信息
- `` ` ``：打开控制台
- `Ctrl+Shift+v`：播放剪贴板里的 URL

如果你是第一次用 MPV，建议先不要急着开补帧、超分、着色器。

先确认普通播放没有问题。

然后按 `Shift+i` 看一下当前视频信息，比如：

- 原始分辨率
- 输出分辨率
- 当前帧率
- 目标帧率
- 是否丢帧
- 当前启用的滤镜

这个信息面板很重要。

因为后面判断补帧、超分有没有生效，基本都靠它。

## 6、补帧怎么用？

补帧主要用的是 RIFE。

简单理解就是：

> 把原本 24 帧、30 帧的视频，通过 AI 生成中间帧，变成更流畅的 48 帧、60 帧或更高帧率。

它对动画观感提升比较明显。

尤其是一些打斗、转场、镜头移动比较多的番剧。

在菜单里可以找到补帧相关选项。

建议新手先从这些模型开始试：

- RIFE 4.15 lite
- RIFE 4.25 lite
- RIFE 4.6
- RIFE 4.6 4K 流畅优先

如果你播放的是 1080P 动画，可以先试 `4.15 lite` 或 `4.25 lite`。

如果是 4K 动画，建议先试轻一点的配置，不要一上来就开最重的模型。

第一次启用某些模型时，可能会弹出命令行窗口进行初始化。

这是正常现象。

等它构建完成后，后续再使用同样配置通常会快很多。

### 6.1 怎么判断补帧是否成功？

按 `Shift+i` 打开详细信息。

重点看这几个地方：

1. 当前帧率是否变高
2. 目标帧率是否符合预期
3. 是否出现持续丢帧
4. 滤镜列表里是否出现 RIFE 相关脚本

如果你看到帧率提升了，但是丢帧一直增加，那说明当前配置对你的机器来说太重了。

这时候可以换轻一点的模型，或者降低处理高度。

如果完全没有变化，可能是滤镜没有成功加载，或者视频本身条件不适合当前脚本。

### 6.2 处理高度是什么？

这次菜单里会看到类似“处理高度”或 `H_Pre` 的选项。

新手可以先这样理解：

> 处理高度就是让 AI 按多高的画面去计算补帧。

比如播放 4K 视频时，如果完整按 2160P 去补帧，压力会非常大。

如果降低处理高度，性能压力会下降，但运动细节也可能变少。

所以它本质上是一个取舍：

- 处理高度高：画面细节更完整，但更吃性能
- 处理高度低：更容易流畅，但运动细节可能下降

如果你是 4K 视频补帧掉帧，可以先试试降低处理高度。

### 6.3 4080S 下 RIFE 性能参考

这次我也专门跑了一批 RIFE 性能测试。

测试环境是我自己的显卡：

```text
RTX 4080 Super
```

测试内容包括：

- 1080P
- 宽屏 4K
- 满高 4K
- 不同 RIFE 模型
- 不同 Turbo 参数
- 不同处理高度

先说结论：

> 1080P 基本压力不大；  
> 宽屏 4K 也比较从容；  
> 满高 4K 才是真正考验配置的场景。

部分测试结果如下，数值是中位 FPS，仅供参考。

| 场景 | 模型 / 参数 | 中位 FPS | 简单说明 |
|---|---:|---:|---|
| 1080P | RIFE 4.6，T1，flow 0.5 | 395.27 | 1080P 下非常轻松 |
| 1080P | RIFE 4.15 lite，T2 | 276.33 | 老牌轻量模型，余量很大 |
| 1080P | RIFE 4.25 lite，T2 | 229.68 | 也很轻松，适合日常测试 |
| 宽屏 4K | RIFE 4.6，T1，flow 0.5，H1440 | 126.42 | 宽屏 4K 下表现很好 |
| 宽屏 4K | RIFE 4.15 lite，T2，H1440 | 104.40 | 仍有明显余量 |
| 宽屏 4K | RIFE 4.25 lite，T2，H1440 | 73.19 | 可用，但余量少一些 |
| 满高 4K | RIFE 4.6，T2，H1440 | 170.36 | 降低处理高度后非常流畅 |
| 满高 4K | RIFE 4.15 lite，T2，H2160 | 60.35 | 满高 4K 原高度下接近可用线 |
| 满高 4K | RIFE 4.25 lite，T2，H2160 | 52.86 | 满高 4K 原高度下更吃力 |
| 满高 4K | RIFE 4.6，T1，flow 0.25，H2160 | 80.17 | 保持 H2160 时相对更稳 |

这张表可以这样看：

- 如果只是 1080P，看数据其实不用太担心性能。
- 如果是宽屏 4K，大多数轻量模型也还比较舒服。
- 如果是满高 4K，建议优先考虑 `4.6 4K 流畅优先` 或者降低处理高度。
- `4.15 lite` 虽然是旧模型，但满高 4K 原高度下还能跑到 60 FPS 左右，仍然有保留价值。
- `4.25 lite` 在部分场景观感不错，但满高 4K 原高度下余量更少。

我还做了一个网页报告，可以更直观看不同模型和参数组合：

https://stupendous-bunny-f65149.netlify.app/

注意，这个结果只代表我的电脑。

不同显卡、驱动、片源、系统环境都会影响实际表现。

但至少可以作为一个初步参考。

## 7、超分怎么用？

超分可以理解为：

> 把低分辨率视频变得更清晰。

这次主要整理了几类超分：

- UAI
- AnimeJaNai
- NVIDIA VSR
- 一些旧版自定义超分配置

如果是 720P 或 1080P 动画，超分可能会比较明显。

如果原片已经是高质量 4K，本身细节已经足够，超分效果可能就没那么明显。

甚至某些真人电影开了超分后，只是显卡占用变高，但主观观感变化不大。

我的建议是：

> 动画低分辨率源可以试超分；  
> 高质量 4K 源优先不要乱开；  
> 补帧和超分不要一上来同时开。

尤其是 4K 超分 + 补帧，性能要求非常高。

即使用高端显卡，也不一定所有片源都能稳。

## 8、着色器怎么用？

着色器主要用于画面增强。

比如：

- 锐化
- 线条增强
- 动画超分
- 去色带
- 降噪
- 缩放优化

这次菜单里对着色器做了一些分类整理。

新手不建议一开始自己乱叠太多。

因为着色器不是越多越好。

叠多了可能会出现：

- 画面过锐
- 线条变脏
- 暗部细节损失
- 动画脸部边缘不自然
- 性能下降

建议先用菜单里的组合。

如果觉得太锐，再手动换轻一点的。

如果觉得没效果，再逐个加。

使用 `Shift+i` 后按相关页面，也可以查看当前正在生效的着色器。

## 9、新手推荐怎么开？

如果你刚开始用，我建议这样：

### 1080P 动画

可以尝试：

- RIFE 4.15 lite
- RIFE 4.25 lite
- 轻量着色器组合
- 必要时开 UAI 或 AnimeJaNai

不建议一开始就补帧 + 超分 + 多个着色器全开。

### 4K 动画

建议优先：

- 只开补帧
- 先试轻量 RIFE
- 掉帧再降低处理高度
- 暂时不要叠重型超分

如果本身就是高质量 4K，超分未必有必要。

### 在线视频

建议先确认：

- 视频能否正常拉起 MPV
- 清晰度是否正确
- 音频是否正常
- 是否需要开启弹幕

在线视频链路比本地视频复杂，先保证播放，再考虑补帧和超分。

## 10、几个常见问题

### 1. 第一次开补帧为什么弹出黑窗口？

可能是在构建模型缓存或 TensorRT engine。

第一次运行需要等待，后面通常会快很多。

### 2. 为什么我开了补帧还卡？

可能是模型太重、视频分辨率太高，或者同时开了超分 / 着色器。

建议先只开补帧，再逐个加其他功能。

### 3. 为什么超分没看出效果？

可能原片本身已经很清晰，或者视频类型不适合当前模型。

超分对低分辨率动画更容易看出效果，对高码率 4K 真人片不一定明显。

### 4. 可以直接覆盖旧版更新吗？

不建议。

最好新建目录使用。

旧版里如果有自己的配置，建议单独备份，再逐项迁移。
