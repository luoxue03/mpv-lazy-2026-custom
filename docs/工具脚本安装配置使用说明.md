# 工具脚本安装、配置与使用说明

本文整理当前整合包中“会被用户直接操作”的安装脚本、MPV 脚本和维护工具。它不是全部源码说明，而是面向安装、配置、使用和排错的速查文档。

## 目录速览

| 路径 | 类型 | 主要用途 | 是否需要用户操作 |
|---|---|---|---|
| `installer/` | Windows 注册/启动辅助脚本 | 注册/取消注册 mpv、安装 umpv、测试/纯净/跑分模式 | 可选 |
| `external_player.js` | 浏览器脚本 | 从网页提取媒体信息并通过 url-scheme-handler 拉起 mpv | 需要浏览器端安装/配置 |
| `portable_config/scripts/` | MPV Lua/JS 脚本 | 菜单、弹幕、清晰度、补帧参数、字幕、历史、Trakt 等 | 多数自动加载，部分需配置 |
| `portable_config/script-opts/` | 脚本选项 | 部分脚本的独立配置 | 需要按功能修改 |
| `tools/` | 维护/测试工具 | RIFE benchmark、engine 预构建、模型导出、Telegram Bridge、Release 打包 | 维护者使用 |

## `installer/`：Windows 注册与启动辅助

`installer/` 应随 `base` 包发布，也保留在仓库中，方便用户下载完整包后自行注册。

### 文件说明

| 文件 | 作用 | 使用方式 | 风险/备注 |
|---|---|---|---|
| `mpv-register.bat` | 调用 `mpv.com --register` 注册 mpv | 双击运行 | 通常用于让系统识别 mpv，可随时取消注册 |
| `mpv-unregister.bat` | 调用 `mpv.com --unregister` 取消注册 | 双击运行 | 卸载或移动目录前可运行 |
| `umpv-install.bat` | 注册 `umpv.exe` 为系统“打开方式”和默认程序候选 | 右键以管理员运行 | 会写入 `HKLM` 注册表，需要管理员权限 |
| `umpv-uninstall.bat` | 移除 `umpv.exe` 注册表项 | 右键以管理员运行 | 用于撤销 `umpv-install.bat` |
| `mpv-测试模式.bat` | 以测试配置启动 mpv | 双击运行 | 用于排查主配置问题 |
| `mpv-纯净模式.bat` | 以尽量少的配置启动 mpv | 双击运行 | 用于判断问题是否来自自定义脚本/配置 |
| `mpv-跑分模式.bat` | 以 benchmark 配置启动 mpv | 双击运行 | 配合性能测试使用 |
| `mpv-BenchMark.conf` | 跑分模式配置 | 被 `mpv-跑分模式.bat` 调用 | 不建议日常播放使用 |
| `mpv-test.conf` | 测试模式配置 | 被 `mpv-测试模式.bat` 调用 | 用于临时诊断 |
| `mpv-icon.ico` | 注册表关联图标 | 被 `umpv-install.bat` 使用 | 不要删除 |

### 推荐使用

- 只想播放视频：不需要运行 installer，直接打开 `mpv.exe` 即可。
- 想把 mpv 加入系统“打开方式”：右键管理员运行 `installer\umpv-install.bat`。
- 想撤销系统关联：右键管理员运行 `installer\umpv-uninstall.bat`。
- 想排查配置是否导致问题：先用 `mpv-纯净模式.bat`，再用 `mpv-测试模式.bat` 对比。


## `portable_config/scripts/`：MPV 自动加载脚本

这些脚本放在 `portable_config/scripts/`，mpv 启动时会自动加载。

### 播放与菜单增强

| 脚本 | 作用 | 配置/入口 |
|---|---|---|
| `uosc/` | 主 UI / 菜单框架 | 主要通过 `input_uosc.conf` 菜单调用 |
| `manager.lua` | 管理器窗口 | 菜单入口 |
| `input_plus.lua` / `inputevent.lua` | 输入增强、快捷键事件支持 | 被其他脚本和菜单间接使用 |
| `recentmenu.lua` | 最近播放菜单 | 快捷键/菜单调用 |
| `save_global_props.lua` | 保存和恢复全局播放属性 | 自动运行 |
| `stats.lua` | 详细统计信息覆盖层 | `I` 或菜单调用 |

### 在线视频与清晰度

| 脚本 | 作用 | 配置/入口 |
|---|---|---|
| `quality-menu.lua` | HLS / yt-dlp 清晰度菜单 | 菜单调用，支持 edition / hls-bitrate 等路径 |
| `ytdl-retry.lua` | yt-dlp 403 自动重试 | 自动运行，主要处理特定网站概率性失败 |

### 补帧、超分与 VF

| 脚本 | 作用 | 配置/入口 |
|---|---|---|
| `rife_runtime_menu.lua` | RIFE 动态参数菜单 | `VF 滤镜 > 补帧 > RIFE 自定义参数` |
| `portable_config/vs/*.vpy` | 补帧、超分、组合滤镜脚本 | 通过 `input_uosc.conf` 菜单调用 |
| `portable_config/script-opts/rife_runtime_default.json` | RIFE 动态菜单默认值 | 可手动编辑，也可由菜单保存 |

### 弹幕与 B 站

| 脚本 | 作用 | 配置/入口 |
|---|---|---|
| `bilibiliAssert/` | B 站弹幕/字幕相关集成 | 配合浏览器拉起和 B 站参数使用 |
| `uosc_danmaku/` | uosc 弹幕插件 | 配置见 `script-opts/uosc_danmaku.conf` |

### 字幕、字体与缩略图

| 脚本 | 作用 | 配置/入口 |
|---|---|---|
| `sub-fastwhisper.lua` | AI 字幕 / whisper 调用 | 配置集中在 `script-opts.conf` |
| `auto_load_fonts.js` | 自动加载字体 | 自动运行 |
| `auto_sub_fonts_dir.lua` | 字幕字体目录辅助加载 | 自动运行 |
| `thumb_engine/` | 缩略图引擎 | uosc / 菜单联动 |
| `thumbfast_2025.lua` | 2025 旧缩略图脚本备份 | 保留参考，不推荐优先使用 |
| `playlist_osd_2025.lua` | 2025 旧播放列表 OSD 备份 | 保留参考，不推荐优先使用 |


### `script-opts.conf`

集中保存大量脚本选项。需要特别注意：

- `sub_fastwhisper-api_key` 默认应为占位符，发布前不能包含真实 key。
- 与个人账号、token、cookie 相关的值不要提交。
- 修改后建议用 `smoke_test_mpv.py` 启动 mpv，看日志是否有脚本配置错误。

### 独立配置文件

| 文件 | 作用 |
|---|---|
| `uosc_danmaku.conf` | uosc 弹幕样式、区域、速度等配置 |
| `rife_runtime_default.json` | RIFE 动态菜单默认参数 |
| `rife_runtime.json` | RIFE 动态菜单当前运行状态，属于运行态，不应提交 |

## `tools/`：维护与测试工具

这些脚本主要面向维护者，不是普通用户日常播放必须项。

### RIFE / TensorRT 性能工具

| 工具 | 作用 | 常见用途 |
|---|---|---|
| `rife_benchmark.py` | 批量测试 RIFE 组合性能 | 生成不同模型、Turbo、分辨率下的 fps 数据 |
| `rife_engine_builder.py` | 预构建 TensorRT engine | 发版前提前构建/验证模型组合 |
| `rife_engine_monitor.py` | 监控 engine 构建过程 | 长时间构建时观察进度和失败项 |

| 工具 | 作用 |
|---|---|
| `export_realesr_general_x4v3_dynamic.py` | 导出 RealESRGAN General x4v3 动态 ONNX，用于实时超分实验 |


## `tools/release/`：发版工具

| 工具 | 作用 |
|---|---|
| `package_release.py` | 生成 `base / ai / config / docs` 分类发布包，并可上传 GitHub Release |

## 推荐排错顺序

### mpv 打不开或系统关联失败

1. 直接双击 `mpv.exe` 测试是否能打开。
2. 用 `installer\mpv-纯净模式.bat` 排除配置问题。
3. 需要系统打开方式时，管理员运行 `installer\umpv-install.bat`。

### 补帧/超分失败

1. 确认完整安装了 `base + ai + config`。
2. 确认根目录存在 `portable.vs`。
3. 用 `tools/release/smoke_test_mpv.py --vf "vapoursynth=~~/vs/xxx.vpy"` 测试具体滤镜。
4. 查看日志里是否有 `VSScript`、`vapoursynth`、`TensorRT`、`trtexec` 相关错误。

### 浏览器拉起失败

1. 确认 `url-scheme-handler.exe` 已正确注册。
2. 确认浏览器用户脚本已启用。
3. 确认 mpv 根目录没有被移动；移动目录后需要重新注册。
4. 对需要登录态的网站，确认 `cookies.txt` 是否可用。

### 发布包验证

1. 运行 `python tools\release\package_release.py`。
2. 运行 `python tools\release\smoke_test_mpv.py --root <解压后的发布目录> --video <本地视频>`。
3. 如果改了 VF 或模型，追加 `--vf "vapoursynth=~~/vs/xxx.vpy"`。
